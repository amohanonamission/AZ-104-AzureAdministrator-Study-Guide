# Exam Tips for Azure Administrator Associate (AZ-104) Candidates

These are the lessons I learned while preparing for and taking the AZ-104 exam. Some are study tips, while others are exam-day strategies that helped me approach Microsoft's questions more effectively.

---

## Before the Exam

### Build a strong foundation

- Spend time inside the Azure Portal.
- Understand **why** Azure services exist instead of memorizing their names.
- Learn how Azure services work together.
- Microsoft Learn is your best resource whenever a topic isn't clear.

---

### Practice the right way

Practice exams helped me the most—but only because I reviewed every mistake.

Don't simply check whether your answer was correct.

Instead ask yourself:

- Why is this the correct answer?
- Why are the other options wrong?
- What Azure service is Microsoft actually testing?

The review process is where most of the learning happens.

---

### Get Hands-on

Reading isn't enough.

Build resources yourself.

Create and delete:

- Virtual Machines
- Storage Accounts
- Virtual Networks
- NSGs
- Azure Monitor
- Azure Backup
- RBAC assignments
- Microsoft Entra ID objects

Deploy → Break → Fix → Delete → Repeat.

---

## Understand the Exam

AZ-104 isn't a difficult exam because the concepts are individually hard.

It's difficult because the syllabus is broad.

You're expected to think like an Azure Administrator who understands:

- Identity
- Networking
- Compute
- Storage
- Governance
- Monitoring
- Backup & Disaster Recovery

The more comfortable you become with Azure terminology, the easier the questions become.

---

## During the Exam

A few strategies that helped me:

- Read the last sentence of the question first.
- Identify the requirement before reading the options.
- Eliminate obviously incorrect answers.
- Flag difficult questions and return later if needed.
- Don't panic if you see unfamiliar wording—break the scenario into smaller pieces.

Microsoft usually isn't testing obscure facts.

They're testing whether you can choose the most appropriate Azure service for a given scenario.

---

## Watch for These Keywords

Microsoft loves asking for the solution that provides:

- Least administrative effort
- Lowest cost
- Most secure
- High availability
- Business continuity
- Scalability
- Azure-native solution
- Minimum downtime

These words often determine the correct answer.

---

## Know These Comparisons Really Well

Many AZ-104 questions are simply comparisons between similar Azure services.

Be comfortable explaining the difference between:

- RBAC vs Azure Policy
- Azure Policy vs Resource Locks
- Service Endpoint vs Private Endpoint
- VPN Gateway vs ExpressRoute
- Load Balancer vs Application Gateway
- Traffic Manager vs Azure Front Door
- Azure Backup vs Azure Site Recovery
- Availability Sets vs Availability Zones
- Blob Storage vs Azure Files
- LRS vs ZRS vs GRS vs RA-GRS
- Managed Identity vs Service Principal

If you can confidently explain each pair without looking at your notes, you're in a strong position.

---

## If You're Stuck Between Two Answers

Ask yourself:

- Which option requires the least administrative effort?
- Which option is the most secure?
- Which option is fully managed?
- Which option is the Azure-native solution?
- Which option satisfies **all** of the requirements?

If two answers are technically correct, Microsoft usually expects the one that best satisfies every requirement in the question.

---

## My Biggest Takeaways

These were the habits that helped me the most:

- Understand concepts instead of memorizing answers.
- Spend time in the Azure Portal.
- Use Microsoft Learn whenever you're unsure.
- Review incorrect practice questions thoroughly.
- Build mental connections between Azure services.
- Think like an administrator solving a business problem—not someone trying to pass a quiz.

---

## Final Thought

Azure administration isn't about memorizing every service or configuration option.

It's about understanding Microsoft's cloud platform, selecting the right solution for each scenario, and building enough hands-on experience to make confident decisions.

Every deployment, mistake, and troubleshooting session makes you a better Azure Administrator.
